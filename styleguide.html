My Fantastic Style Guide

  1. Unassigned functions are not to be named inline.

    JSCS: disallowNamedUnassignedFunctions
    COOL 
    [].forEach(function() {});
var assigned = function() {};
function named() {};
    NOT COOL 
    [].forEach(function namedAndUnassigned() {});

  2. Curly braces are required after the following statements:

    • "if"
    • "else"
    • "for"
    • "while"
    • "do"
    • "try"
    • "catch"
    JSCS: requireCurlyBraces
    COOL 
    if (condition) {
    someThing();
}
    NOT COOL 
    if (condition) someThing();

  3. When expressions involving binary operators are split between lines, the operator must appear before the line break.

    JSCS: requireOperatorBeforeLineBreak
    COOL 
    var x = y ?
        1 : 2;
    NOT COOL 
    var x = y
        ? 1 : 2;

  4. Identifiers must be camelCased or UPPERCASE_WITH_UNDERSCORES

    JSCS: requireCamelCaseOrUpperCaseIdentifiers
    COOL 
    var camelCase = 0;
var CamelCase = 1;
var _camelWithUnderscore = 2;
var UPPER_CASE = 3;
    NOT COOL 
    var lower_case = 0;
var Mixed_case = 2;
var mixed_Case = 3;

  5. All constructors except for this must be capitalized.

    JSCS: requireCapitalizedConstructors
    COOL 
    var x = new A();
var y = new this();
    NOT COOL 
    var x = new a();

  6. When a list is split over multiple lines, the comma separator must come at the end of a line.

    JSCS: requireCommaBeforeLineBreak
    COOL 
    var x = {
    one: 1,
    two: 2,
    three: 3
};
    NOT COOL 
    var x = {
    one: 1
    ,two: 2
    ,three: 3
};

  7. Use $ to begin a variable or object property name which is a jQuery assignment.

    JSCS: requireDollarBeforejQueryAssignment
    COOL 
    var $x = $(".foo");
var y = {
    $x = $(".foo")
};
    NOT COOL 
    var y = $(".foo");
var y = {
    x = $(".foo")
};

  8. Multiple consecutive var declarations must be declared as a comma-separated list with a single var.

    JSCS: requireMultipleVarDecl
    COOL 
    var x = 1, y = 2;
someOtherCode();
var a = 3,
    b = 4;
    NOT COOL 
    var x = 1;
var y = 2;
someOtherCode();

  9. Lines may have a maximum of 160 characters. Tabs (\t) are counted as 4 characters. Comments are allowed to break this rule. Regular expression literals are allowed to break this rule.

    JSCS: maximumLineLength

  10. Block statements must be indented with 4 space characters.

    JSCS: validateIndentation
    COOL 
    if (a) {
    b = c; \\ line is indented with 4 spaces
}
    NOT COOL 
    if (a) {
      b = c; \\ line is indented with 6 spaces
}

  11. Strings must be quoted with double-quotes. The other quote mark may be used only to avoid having to escape.

    JSCS: validateQuoteMarks
    COOL 
    var s = "A String Value";
var s = 'A String containing "quote mark"';
    NOT COOL 
    var s = 'A String Value';

  12. Function parameters must be separated by ", ". Newlines are ignored.

    JSCS: validateParameterSeparator
    COOL 
    doSomething(1, 2, 3);
other(1, 
    2, 
    3);
    NOT COOL 
    doSomething(1,  2,  3) // extra space between parameters

  13. Use concatenation to span a string across multiple lines.

    JSCS: disallowMultipleLineStrings
    COOL 
    var x = "The " +
        "Lazy Dog";
    NOT COOL 
    var x = "The\
        Lazy Dog";

  14. Lines may not contain a mixture of spaces and tab (\t) characacters.

    JSCS: disallowMixedSpacesAndTabs
    COOL 
       x = 1; // \t
   y = 2; // \s\s\s\s
    NOT COOL 
         z = 3; // \t\s

  15. Only one space is allowed between identifiers, keywords, or any other tokens.

    JSCS: disallowMultipleSpaces
    COOL 
    var x = 5;
    NOT COOL 
    var  x = 5;
var x  = 5;
var x =  5;

  16. Don't put spaces between an object key and the following colon (:)

    JSCS: disallowSpaceAfterObjectKeys
    COOL 
    var x = {a: 1};
var y = {
    a: 1
};
    NOT COOL 
    var x = {a : 1};
var y = {
    a : 1
};

  17. Do not put spaces after the following unary operators:

    • "!"
    • "--"
    • "++"
    JSCS: disallowSpaceAfterPrefixUnaryOperators
    COOL 
    var x = !y;
    NOT COOL 
    var x = ! y;

  18. Do not put spaces before () in a function call.

    JSCS: disallowSpacesInCallExpression
    COOL 
    var x = foo();
    NOT COOL 
    var x = foo ();

  19. Do not place the following keywords on a new line:

    • "else"
    JSCS: disallowKeywordsOnNewLine
    COOL 
    } else {
    NOT COOL 
    }
else {

  20. The following keywords must be followed by a space:

    • "if"
    • "else"
    • "for"
    • "while"
    • "do"
    • "switch"
    • "return"
    • "try"
    • "catch"
    JSCS: requireSpaceAfterKeywords
    COOL 
    if (x) {
    x++;
}
    NOT COOL 
    if(x) {
    x++;
}

  21. The following keywords must be preceeded by a space:

    • "else"
    • "while"
    • "catch"
    JSCS: requireSpaceBeforeKeywords
    COOL 
    } else {
    NOT COOL 
    }else {

  22. Do not allow 'sticky' binary operators. The following operators must be preceeded by a space:

    • "="
    • "+="
    • "-="
    • "*="
    • "/="
    • "%="
    • "<<="
    • ">>="
    • ">>>="
    • "&="
    • "|="
    • "^="
    • "+="
    • "+"
    • "-"
    • "*"
    • "/"
    • "%"
    • "<<"
    • ">>"
    • ">>>"
    • "&"
    • "|"
    • "^"
    • "&&"
    • "||"
    • "==="
    • "=="
    • ">="
    • "<="
    • "<"
    • ">"
    • "!="
    • "!=="
    JSCS: requireSpaceBeforeBinaryOperators
    COOL 
    x != y;
    NOT COOL 
    x!= y;

  23. A space is required after all binary operators.

    JSCS: requireSpaceAfterBinaryOperators
    COOL 
    x + y;
    NOT COOL 
    x +y;

  24. In the conditional (ternary) operator, spaces are to be included as shown.

    JSCS: requireSpacesInConditionalExpression
    COOL 
    var x = a ? b : c;
    NOT COOL 
    var x = a?b:c;

  25. A space is required before the opening brace of a block statement.

    JSCS: requireSpaceBeforeBlockStatements
    COOL 
    if (condition) {
    foo();
}
    NOT COOL 
    if (condition){
    foo();
}

  26. A space is required between the clauses of a for statement (after the semicolons).

    JSCS: requireSpacesInForStatement
    COOL 
    for(var i = 0; i < len; i++) {
    x++;
}
    NOT COOL 
    for(var i = 0;i < len;i++) {
    x++;
}

  27. The last line of a source file must end with a line feed.

    JSCS: requireLineFeedAtFileEnd

  28. In an object literal, there must be a space after the key.

    JSCS: requireSpaceBeforeObjectValues
    COOL 
    var x = {a: 1};
    NOT COOL 
    var x = {a:1};

  29. When calling a function, there must be a space after each argument separator.

    JSCS: requireSpaceBetweenArguments
    COOL 
    foo(a, b);
    NOT COOL 
    foo(a,b);

  30. There must be a space before the opening curly brace in a function expression.

    JSCS: requireSpacesInFunctionExpression
    COOL 
    var x = function() {};
    NOT COOL 
    var x = funciton(){};

  31. There must not be a space before the opening paren in a function expression.

    JSCS: disallowSpacesInAnonymousFunctionExpression
    COOL 
    var x = function() {};
    NOT COOL 
    var x = function () {};

  32. Do not put a space before the opening paren in a function expression or definition.

    JSCS: disallowSpacesInFunction
    COOL 
    function a(){};
function a() {};
var x = function() {};
var x = function a(){};
    NOT COOL 
    function a (){};
function a () {};

  33. In an object literal, do not put a space before the closing curly brace.

    JSCS: disallowSpacesInsideObjectBrackets
    COOL 
    var x = {a: 1};
    NOT COOL 
    var x =  {a: 1 };

  34. In an Array, do not put a space after the opening square bracket, or before the closing square bracket.

    JSCS: disallowSpacesInsideArrayBrackets
    COOL 
    var x = [1, 2];
var x = [1, [2, 3]];
    NOT COOL 
    var x = [ 1, 2];
var x = [1, 2 ];
var x = [1, [ 2, 3 ] ];

  35. In an expression, do not put a space after the opening paren, or before the closing paren.

    JSCS: disallowSpacesInsideParentheses
    COOL 
    var x = (3 + 4) * 5;
var x = (3 * (4 + 5)) * 6;
    NOT COOL 
    var x = ( 3 + 4);
var x = (3 + 4 );
var x = (3 + ( 4 * 5) ) / 6;

  36. Do not put multiple empty lines in a row.

    JSCS: disallowMultipleLineBreaks
    COOL 
    var x = 1;

x++; // preceeding line is empty
    NOT COOL 
    var x = 1;


x++; // preceeding two lines are empty

  37. Do not put a line feed before the opening curly brace of block statememts.

    JSCS: disallowNewlineBeforeBlockStatements
    COOL 
    if (condition) {
    x++;
}
    NOT COOL 
    if (condition)
{
    x++;
}

  38. Do not put a comma after the last element in an array or object literal.

    JSCS: disallowTrailingComma
    COOL 
    var x = {
    a: 1,
    b: 2
};
var x = [
    "A",
    "B",
    "C"
];
    NOT COOL 
    var x = {
    a: 1,
    b: 2,
};
var x = [
    "A",
    "B",
    "C",
];

  39. So-called "Yoda conditions" are not allowed. When doing a boolean comparison, the constant, if any, should be on the right

    JSCS: disallowYodaConditions
    COOL 
    if (a === 1) {
    return;
}
    NOT COOL 
    if (1 === a) {
    return;
}
